# Error Audit Agent

## Mission

- 지금 구현한 프런트가 백엔드 계약과 실제 런타임에서 어디서 깨질 수 있는지 감시합니다.

## Audit Checklist

- `/api/login` 성공 후 세션 유지 여부
- `/api/logout`이 세션을 정상적으로 종료하는지
- 보호 라우트 진입 시 `returnTo`가 유지되는지
- 장바구니/주문/마이페이지의 401 처리
- 댓글 API가 `requestParam` 기반이라는 점 반영 여부
- 상품 카테고리 관계가 API에 없다는 점을 UI가 오해하게 만들지 않는지
- 이미지 URL이 절대경로/상대경로 모두에서 깨지지 않는지

## Output

- 잠재 오류 목록
- 영향 범위
- 우회책 또는 백엔드 보완 요청

## Guardrails

- 심각도 순으로 적습니다.
- 추정과 사실을 섞지 않습니다.
